# Data Validation — Outlier Detection

A Jupyter notebook for automatically detecting outliers using statistical distribution fitting.

This pipeline helps you:
- **Identify outliers** in numeric health measurements (e.g., height, weight, blood pressure, cholesterol)
- **Determine the best statistical distribution** that fits each measurement column
- **Calculate valid upper & lower bounds** for flagging suspicious data entries
- **Generate a summary report** (`Distribution_Summary.csv`) with limits and outlier counts

## How It Works

```
dataset.csv
    │
    ▼
[1] Load Data ───────────► Separates numeric vs. categorical columns
    │
    ▼
[2] Visualise Distributions ──► Histograms & boxplots for visual inspection
    │
    ▼
[3] Find Best-Fit Distributions ──► Tests Normal, Logistic, Log-Normal, Weibull
    │                                using distfit + KS-test verification
    ▼
[4] Visualise Fitted Curves ──► Histograms with theoretical distribution overlay
    │
    ▼
[5] Calculate Limits & Outliers ──► Applies sigma-based rules per distribution
    │
    ▼
Distribution_Summary.csv  ◄── Output file with limits and outlier counts
```

## Read the Output

Open `Distribution_Summary.csv` in Excel or any text editor. Each row represents one column from your dataset:

| Column | Meaning |
|---|---|
| `Variable` | Column name |
| `N` | Number of non-null values in the column |
| `ks_D` | KS test statistic; lower values indicate a better distribution fit|
| `ks_p_value` | KS test p-value; higher values indicate better agreement with the fitted distribution |
| `Best_fit_distribution` | Best statistical distribution found (or "No clear fit" if all distribution ks test p-value < 0.05) |
| `Parameters` | Fitted distribution parameters |
| `Lower_Limit` | Lower bound for valid data |
| `Upper_Limit` | Upper bound for valid data |
| `N_Outliers` | Number of data points outside the limits |

## Configuration & Tuning

### `SIGMA_NORM` 

Controls how strict the limits are for **Normal** and **Log-Normal** columns.

### `SIGMA_C` 

Controls how strict the limits are for **Logistic**, **Weibull**, and **No Clear Fit** columns.

### `hard_floors`
Sets biological minimums to filter out obvious typos. For example, a waist measurement of 10 cm is impossible — the hard floor removes it.

## Categorical Columns

Columns that contain text (e.g., gender) are **automatically skipped** from numerical analysis and listed as "Categorical data skipped" in the output.
